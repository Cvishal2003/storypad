# Contributing to StoryPad

Thank you for wanting to help make StoryPad better! Here’s how you can contribute:

## Ways to Contribute

1. **Report Bugs:** [Open an issue](https://github.com/theachoem/storypad/issues) with steps to reproduce. Please include any supporting attachments (such as screenshots, logs, or videos) if available.
2. **Suggest Features:** Create an issue with your idea.
3. **Improve Documentation:** Fix typos or unclear parts - pull requests are welcome!
4. **Code:** Fix bugs, add features, or improve performance. You can pick any bug from the issues list.
5. **Localization:** Translate StoryPad or improve existing translations! See the [Localization Guide](https://storypad.me/contributions/localization/).

## Getting Started

**Steps:**

1. Fork and clone the repo.
2. Create a branch:
   ```sh
   git checkout -b your-feature
   ```
3. Run the app: see the [README](./README.md#%EF%B8%8F-setup--run-) for detailed setup and run instructions.
4. Make your changes.

## Pull Requests

1. Push your branch.
2. Open a pull request on GitHub (target: `develop` branch).
3. Describe what you changed.

## Coding Style

- The recommended code style and formatting are already configured in the VS Code workspace settings.  
  If you use VS Code with the provided settings, your code should automatically meet our standards.
- Otherwise, follow the [Dart style guide](https://dart.dev/guides/language/effective-dart/style).
- Use clear commit messages.
- Run tests before submitting:
  ```sh
  flutter test
  ```

## Code of Conduct

Be friendly and respectful. Let’s make StoryPad welcoming for everyone!

## Need Help?

- Read the [README](./README.md)
- Browse [issues](https://github.com/theachoem/storypad/issues)
- Start a new issue or discussion if you’re stuck.

Thank you for your contribution!
